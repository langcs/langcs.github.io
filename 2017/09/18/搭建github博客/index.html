Hexo+GitHub博客搭建

搭建Node.js环境

在Node.js官网：https://nodejs.org/en/ 下载安装包。
完成安装后，然后Win+R→输入cmd→打开命令提示符；
输入 node -v、npm -v，出现版本号说明Node.js环境配置成功。

搭建Git环境

在Git官网：https://git-scm.com/ 下载安装包。
Next 完成安装。

桌面右键，打开 Git Bush Here，输入 git --version，出现版本号说明Git环境配置成功。

注册和配置GitHub

Github注册：https://github.com/

创建仓库：New repository

yourname/yourname.github.io

注意：仓库命名规则 yourname 必须是你的用户名

访问 yourname.github.io，如果可以正常访问，那么注册和配置Github已经完成。

安装配置Hexo

打开Hexo的官方文档：https://hexo.io/zh-cn/

使用npm安装Hexo：在命令行输入

npm install hexo-cli -g

查看Hexo的版本

hexo version

安装Hexo完成后，请执行下列命令来初始化Hexo，用户名改成你的Hexo将会在指定文件夹中新建所需要的文件。

hexo init yourname.github.io
cd yourname.github.io
npm install

新建完成后，指定文件夹的目录如下：


    ├── .deploy_git     #需要部署的文件  
    ├── _config.yml     #全局配置文件  
    ├── public          #静态网页文件  
    ├── scaffolds       #模板  
    ├── source          #源文件
    |   └── _posts      #文章  
    ├── themes          #主题  
    ├── node_modules    #Hexo插件  
    └── package.json    #npm依赖

运行本地hexo服务

hexo server 或者 hexo s

注意：执行hexo server提示找不到该指令
解决办法：在Hexo3.0后，server被单独出来，需要额外安装server，安装命令如下：
npm install hexo -server –save

你的网站会在http://localhost:4000下启动。
如果http://localhost:4000能够正常访问，说明Hexo本地博客已经搭建完成。

关联Hexo与GitHub Pages

用SSH keys让本地git项目与远程的github建立联系

生成SSH keys

输入你自己的邮箱地址

ssh-keygen -t rsa -C “youremail”

在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入，按回车不设置密码。

添加SSH Key到GitHub

打开C:\Users\Administrator\.ssh\id_rsa.pub，此文件里面内容为刚才生成的密钥，准确的复制这个文件的内容，粘贴到https://github.com/settings/keys的new SSH key中。

输入命令看看设置是否成功

ssh -T git@github.com

配置Git个人信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善。

git config –global user.name “yourname”
git config –global user.email “youremail”

配置_config.yml

在_config.yml文件中，找到Deployment，然后按照如下修改（需要注意冒号后面空一格） 


    # Deployment
    ## Docs: https://hexo.io/docs/deployment.html
    deploy:
        type: git
        repo: git@github.com:yourname/yourname.github.io.git
        branch: master

本地文件提交到Github Pages

//删除旧的public文件
hexo clean

//生成新的public文件
hexo generate 或者 hexo g

//开始部署
hexo deploye 或者 hexo d

在浏览器输入https://yourname.github.io即可浏览。

注意：若上面操作失败，需要提前安装一个扩展：

npm install hexo-deployer-git –save

Hexo的常用操作

发表一篇文章

hexo new “文章标题”

打开本地博客文件夹source\_posts文件夹下可以看到刚刚新建的.md文件；

也可以在source->_deploy文件夹下手动添加.md文件。

文章编辑完成运行生成、部署命令：

hexo clean
hexo g
hexo d

也可以执行下面的命令

hexo clean
hexo d -g

给文章添加多个标签 


    tags: [a, b, c]
    或
    tags:
        - a
        - b
        - c

新建自定义页面

hexo new page folder

显示部分文章内容

如果在博客文章中增加<!-- more -->, 后面的内容就不会显示在列表中了。
